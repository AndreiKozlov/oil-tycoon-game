## 0. Что это вообще

**Концепция в одной фразе:** глобальная карта, ограниченное число участков, на каждом — биом и ресурс, ты строишь экономику от добычи до города-потребителя, конкурируешь с другими игроками за лучшие участки.

**Жанр:** F2P стратегия / экономический симулятор.
**Платформы:** Telegram Mini App + Web → потом iOS/Android.
**Аналог-референсы:** Forge of Empires (ранние эпохи), Travian, Township-экономика, но **с общей картой**.
**Ориентация:** landscape.

**Уникальные фишки:**
1. Общая карта 30 000 тайлов с **реальной редкостью** ресурсов.
2. **Заброшенные участки** возвращаются в оборот — постоянное обновление карты без эпох-сбросов.
3. Разведка с погрешностью по уровню геолога — есть смысл прокачивать.
4. Реальные биржевые цены на ресурсы (этап 3+).

## 2. Карта мира

### 2.1 Размер и форма

- **Карта идентична Земле** (real-world geography). НЕ выдуманная Сибирь — реальный континентальный контур.
- **30 000 покупаемых тайлов** — это только **суша**. Вода (океаны, моря, крупные озёра) присутствует на карте, но **не покупается и в счёт не идёт**.
- Морские тайлы используются только как фон + **возможность поставить морскую платформу** на прибрежном тайле (даёт доступ к шельф-нефти/газу).
- Каждый тайл — пара `(x, y)` в logical-сетке. Изометрия — только при отрисовке.
- Тайлы группируются в **регионы** (континенты + крупные части континентов: Европа, Азия, Северная Америка, Южная Америка, Африка, Австралия, Ближний Восток и т.д., ~10-20 регионов).
- Регион = единица разведки + шардирования на бэкенде.

### 2.1.1 Откуда берётся география

Для генерации используется **публичный GeoTIFF/raster** с биомами Земли (например, [Natural Earth](https://www.naturalearthdata.com/) или [WorldClim](https://www.worldclim.org/)). На бэкенде один раз:
1. Качаем raster биомов в низком разрешении.
2. Накладываем сетку под 30k суши-тайлов (плотнее в населённых регионах, реже в Антарктиде/Гренландии).
3. Для каждого тайла читаем биом из пикселя + докручиваем «потенциал ресурса» (нефть → Ближний Восток/Зап.Сибирь/Аравия, лес → Сибирь/Канада/Амазония и т.д.).
4. Записываем 30 000 строк в Postgres-таблицу `tiles`.

Это **разовая генерация**, делается один раз для прода. На этапе разработки используем уменьшенную версию (~3000 тайлов, чтобы быстрее тестировать).

### 2.2 Биомы тайлов

Биом фиксируется при генерации карты, не меняется. Каждый биом даёт стартовый «потенциал ресурса»:

| Биом | Эмодзи | Цвет | Основной ресурс | Можно строить |
|---|---|---|---|---|
| Лес (тайга) | 🌲 | #1e6e3a | Лес | Лесопилка, охот.хозяйство |
| Лиственный лес | 🌳 | #2d8a4e | Лес + Еда | Лесопилка, ферма (низкая урож.) |
| Степь / поле | 🌾 | #c7b86b | Еда | Ферма, элеватор |
| Горы | ⛰ | #6b6b6b | Камень + Руда | Каменоломня, шахта |
| Тундра | ❄️ | #c8d6dc | Нефть, Газ | Скважина, ГПЗ |
| Пустыня | 🏜 | #d4a857 | Нефть | Скважина |
| Шельф (мелководье) | 🌊 | #4a8ec2 | Нефть, Газ | Морская платформа |
| Болото | 🪵 | #5a6336 | Газ | Газовая скважина |
| Холмы | 🟫 | #7d6b4a | Смешанное (любой ресурс с шансом) | Любое |
| Прибрежная зона | 🏖 | #e5d49a | — (нет ресурса) | Порт, город |
| Равнина | ⬜ | #b8a87c | — | **Любая инфраструктура** (НПЗ, ГПЗ, ТЭЦ, город) |
| Берег реки | 💧 | #5fa3c2 | Еда (рис) | Ферма, ГЭС |

**Бесресурсные тайлы** (равнина, прибрежная, холмы без подсветки) — самые ценные для **инфраструктуры**: НПЗ, ТЭЦ, города. Без них экономика не замыкается.

### 2.3 Распределение биомов

При генерации карты (one-time, серверная):
- 35% — лес/лиственный лес (часто, базовый)
- 15% — степь/поле
- 12% — горы
- 8% — тундра
- 6% — шельф
- 5% — пустыня
- 5% — холмы
- 6% — равнина (мало!)
- 4% — прибрежная зона
- 3% — болото
- 1% — берег реки

Конкретные числа подкрутим на этапе балансинга. **Дефицит равнины** — главный двигатель конфликтов: чтобы построить НПЗ, нужно купить или захватить равнину.

### 2.4 Запасы ресурсов

Кроме биома, каждый тайл с ресурсом имеет **скрытое количество запасов** (Reserves):
- Малые: 50k единиц
- Средние: 200k
- Большие: 800k
- Гигантские: 3M (редко, 1% тайлов)

Распределение запасов — лог-нормальное. Большая часть тайлов — мелкие, гиганты редкие.

**Запасы не известны игроку до разведки.** До разведки виден только биом. После разведки — диапазон с погрешностью (см. п.5).

### 2.5 Серверная истина

Карта — **общая для всех**. State каждого тайла хранится в Postgres:
```
tiles:
  id INT PK
  x, y INT
  region_id INT
  biome ENUM
  reserves_total INT   -- скрыто до разведки
  reserves_remaining INT
  owner_user_id INT NULL   -- null = свободный
  owned_since TIMESTAMP
  last_active_at TIMESTAMP   -- для определения "заброшенный"
  buildings JSONB
```

## 3. Стартовый участок

При первом входе игроку **случайно** даётся один тайл из «стартового пула» (умеренно богатый, биом — `mixed_lowland` или `forest`, чтобы можно было сразу что-то добывать).

Стартовый пул фиксированный (например, ~5000 тайлов помечены `is_starter`). Когда пул кончается — мы переходим на этап 2 расширения карты.

**Стартовая выдача:**
- 1 тайл (биом гарантированно с ресурсом).
- $50 000 игровых.
- 5 кристаллов (премиум).
- 1 разведка бесплатно (для разведки соседнего тайла).
- Туториал: Геолог Иван объясняет первые шаги.

### 4.2 Базовые цепочки на этапе G

**Простая цепочка (для туториала):**
```
Лес-тайл → Лесопилка → 🪵 → Продажа на бирже → 💰
```

**Замкнутая цепочка (для опытного игрока):**
```
Лес-тайл → Лесопилка → 🪵 ↘
Гора-тайл → Каменоломня → 🪨 ↘
Поле-тайл → Ферма → 🌾  →→→ Город (потребляет всё) → 💰 налоги
Прибрежье → Ветряк → ⚡ ↗
```

Игрок сначала зарабатывает на продаже сырья, потом строит свой первый Город — главная цель этапа G.

## 5. Разведка

Перед покупкой тайла можно сделать **разведку** — узнать примерные запасы.

Технология «Геология» имеет уровни:
| Уровень | Точность данных |
|---|---|
| 0 (нет) | Только биом. Запасы неизвестны. |
| 1 | ±50% к запасам |
| 2 | ±30% |
| 3 | ±15% |
| 4 | ±5% |
| 5 | Точно |

Разведка стоит небольших денег ($1k-$10k за тайл, растёт с уровнем игрока).
Кристаллы → мгновенная точная разведка одного тайла.

## 6. Монетизация

### 6.1 Премиум-валюта: кристаллы 💎

Покупка за реальные деньги через Telegram Stars (на старте) и ЮKassa/Stripe (на этапе 4).

### 6.2 На что тратятся кристаллы

- **Ускорить исследование/строительство** (10 💎 за 1 час).
- **Точная разведка** тайла (5 💎).
- **Мгновенная покупка** соседнего тайла (если денег не хватает — конвертация $→💎).
- **Открыть слот** на участке (постройка +1).
- **Защита от захвата** заброшенного участка (3 дня дополнительной грейс-периодицы за 20 💎).

### 6.3 Что НЕ за кристаллы (anti-P2W)

- Боевая мощь / захват у активных игроков — **никак** (PvP отложен или вообще нет).
- Эксклюзивные участки — нет.
- Получение ресурсов «из воздуха» — нет (только ускорение производства).

## 7. Заброшенные участки

Если игрок не заходит **N дней** (стартово N=14, подкрутим), его участки помечаются «заброшенными»:
- Зданий не работают (производство 0).
- Через ещё 7 дней неактивности — можно «занять» (с дисконтом 50% к рыночной цене покупки).
- При занятии: постройки **сохраняются**, но 30% повреждены — требуют починки ресурсами.

Это решает проблему «все купили, новички не имеют куда расти».

---

## 8. Прогрессия игрока

- **Уровень игрока** определяется суммарным XP за весь аккаунт.
- XP даётся за: продажу ресурсов, заключённые сделки, доход от Города.
- **HQ-уровень** = `floor(level / 3) + 1`. HQ определяет:
  - Сколько участков можешь владеть параллельно (1, 2, 4, 8, 16, 32).
  - Сколько построек на каждом (8, 10, 12, 14, ...).

---

## 9. Технологическое дерево

5 веток:
1. **Геология** — точность разведки.
2. **Промышленность** — скорость переработки, новые рецепты.
3. **Энергетика** — КПД ТЭЦ, открытие ГЭС и АЭС.
4. **Логистика** — дальность торговли, дороги, разгрузка цепочек.
5. **Урбанизм** — потребление Города, налоговый доход.

Каждая ветка — 5 уровней. Цена и время растут экспоненциально. Все ветки исследуются в одном инстансе HQ.

## 10. UI / Layout

> **Решение 2026-05-18 (после теста G.2):** отказались от постоянных боковых панелей в пользу **fullscreen-сцены с floating overlays** (стандарт Last Day on Earth / Township / Clash of Clans). В игре **два разных экрана** с разным контекстом.

### 10.0 Общие принципы UI (для всех экранов)

- **Landscape only.** В portrait на телефоне — приглашение «📱➡️ поверни телефон».
- **Сцена занимает 100% экрана.** UI поверх как floating-чипы и кнопки.
- **HUD сверху** — постоянно: гамбургер ☰ + имя слева, $/💎/⚡/XP справа. ~40px высоты.
- **Bottom-bar ресурсов** — постоянно снизу. 5-6 чипов адаптированных под текущий экран. По тапу разворачивается полный список всех ресурсов. ~32px высоты.
- **Меню навигации** — drawer слева, открывается по ☰. Вкладки: Карта мира, Локация, Биржа, Наука, Рейтинг.
- **Уведомления-баннеры** — всплывают над сценой, исчезают через 5 сек. Примеры: «Лесопилка заполнена», «Исследование готово», «Город платит налоги».

### 10.4 Уведомления-баннеры

Появляются в верхней трети экрана, по центру, под HUD-чипами. Длительность: 5 секунд. Стиль:

```
   ┌─────────────────────────────────┐
   │ 🪵 Лесопилка заполнена · 100   │
   │    Собери чтобы продолжить     │
   │              [Собрать]          │
   └─────────────────────────────────┘
```

**События, которые показывают баннер:**
- Здание заполнено (нечем продолжать копить).
- Исследование завершено.
- Город платит налоги (раз в час).
- Атака / захват заброшенного участка (этап G.8).
- Уровень повышен (level up — отдельный баннер с искрами).

## 11. Архитектура (этап G.4)

### 11.1 Бэкенд

- **Fastify** на Node 20.
- **Postgres 16** — постоянное хранение (tiles, users, buildings, resources).
- **Redis 7** — кэш видимых регионов карты, сессии, rate-limit.
- **Telegram auth** — валидация `initData` HMAC-SHA256 с bot-token.
- **WebSocket** для real-time обновлений (захват тайла, налоги Города).

### 11.2 Шардирование карты

- Карта разбита на ~30 регионов. Каждый регион в Redis как Geo-hash.
- Клиент загружает только **видимый regiond** (1-4 региона на экране).
- При pan → подгружаются соседние регионы.

### 11.3 Tick на сервере

- Один глобальный tick раз в N секунд (10s? настроим). Обходит активные постройки, накапливает ресурсы.
- На клиенте — оптимистичный счётчик между серверными тиками.
