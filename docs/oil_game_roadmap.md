# RoadMap: Симулятор добычи нефти и газа

> Версия: 0.2 (мультиплатформенный план)
> Дата: 2026-05-17
> Формат: классическая мобильная игра с донатами, без токена/крипты на старте.
> Платформы: Telegram Mini App → Web → iOS → Android (по очереди).

---

## Принципы по которым строим план

1. **Сначала проверка интереса, потом большая разработка.** Не пилим 6 месяцев в стол.
2. **Каждый этап имеет понятный результат и решение "идём дальше / стоп".**
3. **Минимум денег до первой выручки.** Всё что можно — делаем руками и нейросетями.
4. **Тех. долг копим осознанно.** На MVP — упрощения. На масштабе — переделка.
5. **Один бэкенд — много клиентов.** Архитектура с первого дня готова к мультиплатформенности.

---

## Архитектура: один сервер — много клиентов

```
                ┌─────────────────────────────┐
                │      БЭКЕНД (единый API)    │
                │  Node.js + PostgreSQL +     │
                │  Redis + WebSocket          │
                └──────────────┬──────────────┘
                               │
        ┌──────────────┬───────┴───────┬──────────────┐
        │              │               │              │
   ┌────▼─────┐  ┌─────▼─────┐  ┌──────▼─────┐  ┌────▼─────┐
   │ Telegram │  │ Web (PWA) │  │ iOS app    │  │ Android  │
   │ Mini App │  │ browser   │  │ App Store  │  │ Play     │
   │ (старт)  │  │ (этап 2)  │  │ (этап 4+)  │  │ (этап 4+)│
   └──────────┘  └───────────┘  └────────────┘  └──────────┘
```

**Что это даёт:**
- Один аккаунт — игрок играет с любого устройства, прогресс синхронизирован.
- Логика, баланс, экономика — **только на сервере**. Клиенты — это «окна» в один мир.
- Новая платформа = новый клиент к тому же API, без переписывания игры.

**Технически:**
- Бэкенд один: REST API + WebSocket для реал-тайма.
- Клиенты на одном движке (React + Pixi.js / Phaser) — один кодбейз, разные обёртки:
  - Telegram Mini App = React в Telegram WebView
  - Web = тот же React на сайте
  - iOS/Android = React Native или Capacitor обёртка над тем же React-кодом

**Принцип:** не пишем 4 раза одну игру. Пишем **одну игру** и **4 обёртки**.

---

## Общая структура: 7 этапов

```
ЭТАП 0  Дизайн-документ                    (2-3 недели, $0)
ЭТАП 1  Прототип механики (Web + Telegram) (3-4 недели, $0-100)
ЭТАП 2  MVP мультиплеер (Telegram + Web)   (6-8 недель, $200-500)
ЭТАП 3  Закрытый бета-тест                 (2-3 недели, $100-300)
ЭТАП 4  Открытый запуск + донаты           (постоянно, $500-2000/мес)
ЭТАП 5  Выход в App Store / Google Play    (2-3 месяца, $4-15k)
ЭТАП 6  Масштабирование и live-ops         (постоянно)
```

**Почему именно такой порядок:**
- **Сначала Telegram** — нулевой порог входа, виральность через реферальные ссылки, моментальные платежи через Telegram Stars, не нужно юр.лицо.
- **Параллельно Web** — та же игра в браузере, для SEO и поискового трафика, для тех у кого нет Telegram.
- **Потом App Store / Google Play** — когда есть выручка на юр.лицо, отшлифованная игра и комьюнити с отзывами.

## ЭТАП 0 — Дизайн-документ (Game Design Document)

**Цель:** на бумаге описать всю игру до строчки кода. Если не можем описать — не можем сделать.

**Результат этапа:** папка документов на 20-30 страниц.

### Шаги:
1. **Vision** — что за игра, для кого, чем отличается (1 страница)
2. **Core Loop** — что игрок делает в первые 60 сек, 10 мин, день, неделю, месяц (2 страницы)
3. **Список механик** — добыча, апгрейды, исследования, события, рейтинги, кланы (3-4 страницы)
4. **Экономика в Excel** — стоимость вышек, доход в секунду, время окупаемости, кривая прогресса (отдельный файл)
5. **Монетизация** — где и сколько игрок платит, что продаём, ценники (2 страницы)
6. **UI/UX скетчи** — рисунки от руки или в Figma всех экранов (5-10 экранов)
7. **Конкурентный анализ** — Oil Tycoon, Idle Miner, Mining Empire: что у них работает (3 страницы)
8. **Технический стек и архитектура** — на чём пишем, где хостим, как масштабируем под 4 платформы (2 страницы)

**Бюджет:** $0
**Решение в конце:** идём в код или нет.

## ЭТАП 1 — Прототип механики (Web + Telegram, без сервера)

**Цель:** доказать что **гейм-луп работает**. Можно ли в эту игру играть час и не заскучать.

**Результат этапа:** одна HTML-страница, открывается и в браузере, и в Telegram, играет 1 игрок сам с собой.

### Шаги:
1. **Стек:** React + TypeScript + Pixi.js или Phaser (2D)
2. **Telegram WebApp SDK** подключён, но игра работает и без него (открыл в Chrome — играет)
3. Базовый экран: 1 участок земли, 1 вышка, кнопка «добывать»
4. Реализация core loop: добыча → продажа → апгрейд → быстрее добыча
5. 5-7 уровней апгрейдов вышки
6. 3-5 видов оборудования
7. Сохранение прогресса в localStorage (потом мигрируем на сервер)
8. Тест на себе: играю 1 час подряд, не скучно? Если скучно — переделываем механику.
9. Тест на 3-5 знакомых: дают честную обратную связь.

**Бюджет:** $0-100 (только хостинг для теста, можно бесплатно на Vercel/Netlify)
**Решение в конце:** работает ли механика, нравится ли играть.

## ЭТАП 2 — MVP с сервером (Telegram + Web)

**Цель:** превратить прототип в **сетевую игру** с общим миром, рейтингами и сервером.

**Результат этапа:** полноценная игра в Telegram **и** на сайте, играет 50-500 человек, один аккаунт работает с обеих платформ.

### Шаги:
1. **Бэкенд:** Node.js (TypeScript) + Express/Fastify + PostgreSQL + Redis
2. **WebSocket** для real-time событий (цена нефти, лидерборд, чат)
3. **Авторизация:**
   - Через Telegram WebApp (initData) — для Telegram-клиента
   - Через email/пароль или Google OAuth — для Web-клиента
   - Один аккаунт, обе авторизации могут привязываться к нему
4. **База данных:** игроки, участки, вышки, ресурсы, транзакции, события
5. **API:** добыча, апгрейд, покупка, продажа, лидерборд
6. **Anti-cheat:** вся логика на сервере, клиент только показывает. Никаких «добыто 1000000» с клиента.
7. **Глобальный лидерборд** — топ-100 игроков, обновление в реальном времени
8. **Карта мира** — упрощённая, 100-500 участков, видны соседи
9. **Мировые события** — раз в 6 часов событие меняющее цену нефти (скачок, авария, тендер)
10. **Базовая графика** — генерируем в Midjourney/Flux, собираем UI в Figma
11. **Telegram-бот** для оповещений (нашли нефть, событие, апгрейд готов)
12. **Web-версия (PWA)** — устанавливается как «приложение» из браузера, работает офлайн-просмотр
13. **Деплой:** VPS (Hetzner, $20-50/мес), доменное имя, SSL

**Бюджет:** $200-500 (хостинг + домен + покупка пары готовых ассетов)
**Решение в конце:** запускаем закрытый бета-тест.

## ЭТАП 3 — Закрытый бета-тест

**Цель:** найти баги, проверить ретеншн, отшлифовать UX.

**Результат этапа:** игра выдерживает 500-1000 одновременных пользователей, retention D7 > 15%.

### Шаги:
1. Набор 100-300 тестеров через Telegram-каналы (бесплатно, в свои чаты)
2. Отдельный Telegram-чат для фидбека
3. Метрики: DAU, retention D1/D3/D7, среднее время сессии, частота сессий, конверсия в платящих
4. Исправление топ-20 жалоб
5. Балансировка экономики по реальным данным (а не из Excel)
6. Стресс-тест сервера на 1000 одновременных
7. Финальная вычитка UX до запуска
8. Тестирование Web-версии отдельно (часто баги отличаются от Telegram-версии)

**Бюджет:** $100-300 (платная реклама в 1-2 каналах для теста)
**Решение в конце:** D7 > 15% → открытый запуск. < 5% → переделываем механику. Между — улучшаем.

## ЭТАП 4 — Открытый запуск + донаты (Telegram + Web)

**Цель:** первые деньги. Включаем магазин, рекламу, реферальную программу.

**Результат этапа:** $500-5000 в месяц выручки.

### Шаги:
1. **Магазин внутри игры** — пакеты валюты, премиум-вышки, ускорения
2. **Платежи:**
   - Telegram Stars (для Telegram-клиента, ~30% комиссия)
   - Stripe / ЮKassa / Cloudpayments (для Web, ~3-5% комиссия — если есть юр.лицо)
3. **Реферальная программа** — пригласи друга, оба получают бонус
4. **Ежедневные награды** — заходи каждый день, получай бонус
5. **Первый платный ивент** — соревнование на неделю с призами
6. **Первая реклама** — закуп в 3-5 Telegram-каналов ($300-1000)
7. **Аналитика** — Amplitude / Mixpanel / своя простая аналитика
8. **A/B тесты ценников** — найти оптимальную цену первой покупки
9. **Юр.оформление** — самозанятый/ИП в РФ или ИП в Казахстане/Армении для приёма платежей легально

**Бюджет:** $500-2000 в месяц (маркетинг)
**Решение в конце:** окупается ли реклама (CAC < LTV). Если да — идём в App Store.

## ЭТАП 5 — Выход в App Store и Google Play

**Цель:** охватить аудиторию которая не пользуется Telegram, особенно США/Европу.

**Результат этапа:** игра в обоих магазинах, +10-100k установок в месяц.

### Почему это отдельный большой этап
- Apple строго проверяет первый билд от нового разработчика
- Нужно юр.лицо в подходящей юрисдикции (РФ — заблокировано санкциями для Apple)
- Любые упоминания крипты/токенов/обмена внутриигрового на реальные деньги = **бан**
- Стоит дороже чем кажется: юр.лицо + аккаунты + локализация + маркетинг

### Шаги:
1. **Регистрация юр.лица** в подходящей юрисдикции:
   - Казахстан/Армения/Сербия — $500-2000, быстро, для русскоязычных
   - ОАЭ/Сингапур — $5-15k, для глобального масштаба
2. **Открытие банковского счёта** под юр.лицо
3. **Регистрация developer-аккаунтов:**
   - Apple Developer Program: $99/год
   - Google Play Console: $25 однократно
4. **Сборка нативного клиента:**
   - React Native или Capacitor поверх существующего React-кода
   - Тот же бэкенд, тот же геймплей, нативная обёртка
5. **Адаптация под гайдлайны магазинов:**
   - Убираем все упоминания крипты, TON, токенов, заработка
   - Платежи через Apple IAP / Google Billing (30% комиссия Apple/Google)
   - Контентная политика: возрастной рейтинг, политика конфиденциальности, terms of service
6. **ASO (App Store Optimization):**
   - Название, иконка, скриншоты, описание, ключевые слова
   - Локализация на 5-10 языков (русский, английский, испанский, португальский, турецкий, индонезийский, арабский, хинди и т.д.)
7. **Первый билд → ревью Apple/Google → исправления → релиз**
8. **Платная реклама** в Google Ads, Facebook/Instagram Ads, TikTok Ads, ведущая на магазины
9. **PR и блогеры** — обзоры от мобильных гейм-блогеров

**Бюджет:** $4-15k (юр.лицо + аккаунты + локализация + первый маркетинг)
**Решение в конце:** окупается ли каждый канал привлечения по CAC.

### Что НЕЛЬЗЯ показывать в App Store / Google Play
- Слова «токен», «криптовалюта», «P2E», «play-to-earn», «NFT», «майнинг» (в крипто-смысле)
- Связь с TON, обещания листинга, обмен внутриигровой валюты на реальные деньги
- Ссылки на внешние сайты для оплаты в обход IAP
- Реклама любых сторонних приложений

Если когда-то будет токен (этап 6+) — это **отдельная версия игры**, которая живёт только в Telegram. В App Store — «чистая» версия без крипты.

## ЭТАП 6 — Масштабирование и live-ops

**Цель:** перейти из «проект на коленке» в «настоящий продукт».

**Результат этапа:** команда, юридическая структура, серьёзная выручка ($10k+/мес).

### Шаги:
1. **Live-ops:**
   - Сезоны — раз в 3 месяца новый сезон, новые цели, награды
   - Кланы / союзы — социальная механика
   - PvP-ивенты — соревнования между игроками
   - Новые ресурсы — газ, уголь, литий
   - Расширение карты — новые регионы мира
   - Истории / квесты — нарративный контент
2. **Команда:**
   - Комьюнити-менеджер
   - Второй разработчик
   - Гейм-дизайнер на ивенты
3. **Инфраструктура:**
   - CDN (Cloudflare)
   - Балансировщик нагрузки
   - Реплики БД
   - Мониторинг (Grafana, Sentry)
4. **Бизнес:**
   - Юрлицо в правильной юрисдикции (если ещё нет)
   - Финансовая отчётность, P&L
   - Возможный инвестиционный раунд
5. **Опционально (только когда уже есть выручка $50k+/мес):**
   - Внутриигровой токен на TON (с юристами и аудитом)
   - Партнёрство с TON Foundation
   - Гранты на расширение

**Бюджет:** реинвестиция выручки

## Чего НЕТ в этом плане (важно)
